# 🌲 CMU Green Path - เกมเดินทางแบบอัจฉริยะ

> **นักศึกษาต้องศึกษาอัลกอริทึม Dijkstra โดยเล่นเกมวางแผนเส้นทาง**

---

## 📖 บทนำ

**CMU Green Path** เป็นเกมการศึกษาที่ออกแบบมาเพื่อสอนหลักการของอัลกอริทึม **Dijkstra's Algorithm** ในบริบทที่สนุกและเข้าใจง่าย ผู้เล่นจะบทบาทเป็นนักศึกษาที่ต้องวางแผนเส้นทางการเดินทางจากจุดเริ่มต้นไปยังเป้าหมายในแผนที่มหาวิทยาลัยเชียงใหม่ โดยพิจารณาปัจจัยสองอย่าง:
- ⏱️ **เวลาการเดินทาง (Time)**
- 🌱 **การปล่อยก๊าซคาร์บอนไดออกไซด์ (Carbon Footprint)**

เกมจะเปรียบเทียบการเลือกของผู้เล่นกับสองเส้นทางที่ค้นหาโดย AI โดยใช้อัลกอริทึม Dijkstra เพื่อหาเส้นทางที่เหมาะสมที่สุดในแต่ละด้าน

## 🧮 Dijkstra Algorithm คืออะไร?

### หลักการพื้นฐาน

**Dijkstra's Algorithm** เป็นอัลกอริทึมที่ใช้หาเส้นทางที่สั้นที่สุด (Shortest Path) จากโหนดเริ่มต้นไปยังโหนดอื่นๆ ในกราฟ ถูกค้นพบโดย **Edsger W. Dijkstra** ในปี 1956

### วิธีการทำงาน

1. **เลือกโหนดเริ่มต้น** - กำหนดระยะทาง = 0 โหนดอื่นๆ = ∞
2. **หาโหนดที่ใกล้ที่สุดที่ยังไม่ได้ตรวจสอบ** - เลือกโหนดที่มีระยะทางน้อยที่สุด
3. **อัปเดตระยะทาง** - ตรวจสอบเพื่อนบ้านและอัปเดตระยะทางหากพบเส้นทางที่สั้นกว่า
4. **ทำเครื่องหมายว่าเสร็จสิ้น** - ยืนยันว่าระยะทางถึงโหนดนี้คือค่าที่เหมาะสมที่สุด
5. **ทำซ้ำ** - จนกว่าจะมาถึงเป้าหมายหรือทำเสร็จทุกโหนด

### ตัวอย่างแบบง่าย

```
      A (เริ่มต้น)
     / \
   2/   \5
   /     \
  B       C
  |\ 3    |
  | \     |1
  |1 \    |
  D   E---F (เป้าหมาย)
      2
```

เส้นทางที่สั้นที่สุด: A → B → E → F (ระยะทาง = 2+3+2 = 7)

## 🎮 วิธีการเล่น

### 🎯 เฟสที่ 1: วางแผนเส้นทาง (Planning Phase)

1. **เลือกพาหนะ** 🚴‍♂️
   - 🚶 **เดินเท้า** - ช้ามาก แต่คาร์บอนต่ำสุด

   - 🚐 **รถม่วง (Shuttle)** - เร็ว แต่ลงได้เฉพาะสถานีเท่านั้น
   - 🏍️ **มอเตอร์ไซค์** - เร็วที่สุด แต่คาร์บอนสูง

## 🔗 การเชื่อมโยงกับ Dijkstra

เกมนี้ใช้ **Dijkstra's Algorithm** ในสามวิธี:

### 1️⃣ คำนวณ Eco Route (ค่าใช้จ่ายต่ำสุด = คาร์บอน)
```
Cost = Carbon Footprint ของแต่ละเส้นทาง
Dijkstra หาเส้นทางที่ Carbon น้อยที่สุด
```

### 2️⃣ คำนวณ Express Route (ค่าใช้จ่ายต่ำสุด = เวลา)
```
Cost = Time เดินทาง / Speed ของพาหนะ
Dijkstra หาเส้นทางที่เวลาน้อยที่สุด
```

### 3️⃣ ตรวจสอบว่าเส้นทางของคุณสมเหตุสมผล
```
ระบบคำนวณโดย Dijkstra และแสดงค่า
เพื่อให้ผู้เล่นเห็นผลการเลือกสินค้า
```

## ⚙️ องค์ประกอบพื้นฐาน

### 1. **Graph (กราฟ)**
- **โหนด (Vertices)**: อาคารต่างๆ ในมหาวิทยาลัย
- **เส้นเชื่อม (Edges)**: เส้นทางระหว่างอาคาร
- **น้ำหนัก (Weights)**: เวลา หรือ คาร์บอน

### 2. **Cost Function (ฟังก์ชันต้นทุน)**
```
Cost = (ระยะทาง / ความเร็ว) + (อัตราปล่อยคาร์บอน × ระยะทาง)
```

### 3. **Priority Queue**
อัลกอริทึม Dijkstra ใช้ Priority Queue เพื่อจัดเรียงโหนดตามระยะทาง
