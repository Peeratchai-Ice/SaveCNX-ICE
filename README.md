🎨 1. หน้าตาและกราฟิกของเกม (Visual Appearance)
เกมจะนำเสนอในรูปแบบ 2D Top-Down (มุมมองจากด้านบน) สไตล์ Vector Clean หรือ Minimalist ที่ดูสะอาดตาและทันสมัย คล้ายกับแอปพลิเคชันแผนที่นำทางอัจฉริยะ

ฉากหลัง (The Map): แผนที่จำลองภายในมหาวิทยาลัยเชียงใหม่ มีเส้นถนน, พื้นที่สีเขียว, และจุดสังเกตหลัก

จุดแวะพัก (Nodes): อาคารเรียนต่างๆ จะถูกแทนที่ด้วยไอคอน 2D ที่ดูปุ๊บก็รู้ปั๊บ เช่น ตึกคณะวิศวกรรมศาสตร์, สำนักหอสมุด, หรือศาลาธรรม โดยมีสถานีรถม่วง (Shuttle Station) ที่หน้า มช. และอ่างแก้วเป็นจุดเด่น

เส้นทาง (Paths): เมื่อผู้เล่นลากเส้นวางแผน จะมีเส้นประ (Line Renderer) ปรากฏขึ้น โดยสีของเส้นจะเปลี่ยนไปตามประเภทพาหนะ เช่น สีเขียว(เดิน), สีฟ้า(จักรยาน), สีม่วง(รถไฟฟ้า), สีแดง(มอเตอร์ไซค์)

หน้าจอแสดงผล (UI/UX): * ระหว่างเล่นจะมี Planning HUD เป็นปุ่มให้เลือกพาหนะ

ตอนจบด่านจะมี Result Screen เป็นแผง Dashboard สรุปผลข้อมูล (Data Visualization) เทียบกันแบบ 3 คอลัมน์ที่ดูเป็นมืออาชีพ

🎮 2. ระบบการเล่น (The Gameplay Loop)
ผู้เล่นจะรับบทเป็นนักศึกษาที่ต้องบริหารจัดการ "เวลา (Time)" และ "คาร์บอนฟุตพริ้นท์ (Carbon Footprint)" ในการเดินทางจากจุดเริ่มต้นไปสู่เป้าหมาย โดยแบ่งการเล่นออกเป็น 3 เฟสหลัก:

เฟสที่ 1: วางแผนเส้นทาง (Planning Phase)
ตัวละครจะยืนสแตนด์บายอยู่ที่จุดเริ่มต้น (เช่น หน้า มช.) ผู้เล่นจะต้องวางแผนการเดินทางทีละท่อน (Node to Node)

เลือกพาหนะ: กดปุ่มเลือกวิธีเดินทาง (เดิน, จักรยาน, รถม่วง, มอเตอร์ไซค์)

กำหนดจุดหมาย: คลิกลากเส้นไปยังตึกถัดไป

เงื่อนไขพิเศษ: ถ้านั่งรถม่วง ระบบจะบังคับให้คลิกไปลงได้เฉพาะตึกที่เป็น "สถานี" เท่านั้น (วาร์ปข้ามโซน)

ผู้เล่นจะต่อเส้นทางไปเรื่อยๆ จนกว่าจะลากไปถึงจุดหมายปลายทาง (Goal) เช่น ลานอ่างแก้ว

เฟสที่ 2: ออกเดินทางอัตโนมัติ (Execution Phase)
เมื่อเส้นทางเชื่อมถึงเป้าหมายแล้ว ปุ่ม "GO! (เริ่มเดินทาง)" จะปรากฏขึ้น

พอกดปุ่ม ผู้เล่นจะเปลี่ยนสถานะเป็นผู้ชม (Observer)

ตัวละครจะเลื่อนไปตามเส้นทางที่วางแผนไว้อัตโนมัติ (Auto-Routing)

ความเร็วในการเคลื่อนที่บนหน้าจอ จะช้าหรือเร็วอิงตามสถิติความเร็ว (Speed) ของพาหนะในท่อนนั้นๆ จริงๆ

เฟสที่ 3: บทสรุปแห่งการเรียนรู้ (Evaluation Phase)
ทันทีที่ตัวละครเดินถึงอ่างแก้ว เกมจะหยุดและเด้งหน้าต่าง Report Dashboard ขึ้นมา "เบิกเนตร" ผู้เล่น โดยสมองกลจะรันอัลกอริทึม Dijkstra เพื่อเปรียบเทียบผลลัพธ์ 3 รูปแบบ:

เส้นทางของคุณ (Player's Path): สรุปว่าคุณใช้เวลาไปกี่นาที และสร้างมลพิษไปกี่กรัม

🌱 Eco Route (AI แนะนำ): เส้นทางที่ปล่อยมลพิษน้อยที่สุดในแมพ (คาร์บอนต่ำสุด)

⚡ Express Route (AI แนะนำ): เส้นทางที่ไปถึงอ่างแก้วไวที่สุด (เวลาต่ำสุด)

💡 จุดเด่นที่ทำให้เกมนี้ไม่ธรรมดา (Core Value)
นิยามของเกมนี้ไม่ใช่แค่เกมหาทางออกธรรมดา แต่มันคือ Educational Strategy Simulator การตัดระบบแอ็กชันฟิสิกส์ออกไป แล้วใส่ระบบ Data-Driven (เวลา vs คาร์บอน) เข้ามาแทน ทำให้ตัวโปรเจกต์มีน้ำหนัก มีความสมจริง และแสดงศักยภาพในการเขียนโปรแกรมจัดการโครงสร้างข้อมูลได้อย่างยอดเยี่ยมครับ
